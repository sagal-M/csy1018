Site Report

Introduction

This report is about my experiences this module with learning the basics of web development, coding, design and debugging process. Overall I have enjoyed developing a knowledge and better understanding in each of the different areas within web development. As a beginner, learning even the basics of code was very challenging, because I have not done much before. Through research and troubleshooting, I slowly grew to understand them, but I know there is still much more that I could learn.

By using a logical and step-by-step approach, I have been surprised at what I have been able to produce with my Portfolio site.

Week one and two

Firstly, I started with learning the basic of using html and understanding that every web page uses HTML. The code editor that I used for html was Visual studio code which I was able to get the hand of quite easily. Next I then moved onto learning css and about how html and css both link. After attempting the tasks for both html and css I was able to see exactly what they both do, with only using html I realised that the browser make assumptions about how the site should look however once you add a css file you can override this behaviour and describe how these elements should look, this was very helpful to know when making my site.

Week three and four

After fully understanding the purpose of both html and css, I then moved onto learning how to make web pages responsive which is very useful as I want to be able to make my site mobile friendly, when making a web page responsive I was able to learn some tips on how to make the web page fit on mobile phone screen, for example, I now know how to create a hamburger icon so that everything on the page is not squished together. I also attempted some activities on creating animations with transitions and different hover effects.

Week five

Once I had received the brief for creating my portfolio site I was able to begin the process of creating all the different pages for my site. I started with creating a folder which will have all of the files that will be needed for my portfolio site , I then started to created the home page for the site which will be the first page that people see when they enter my site, I had to carefuly think about the layout and the colours, I decided to use pink as my main themed colour for my site this is because it is a portfolio about myself and my work and pink if one of my favourite colours, i first chose a very bright pink however once I saw how it looked I decided to go for a more subtle pink as it was very distracting, one problem that I had with my first page was that the changes I was making on my css file was not reflecting on my home page however I then realised that this was because I did not link the css file to the html file. Before moving on to a different page I had to create a navigation bar at the top of the page so that it can be easy to move from page to page this is going to be on each page I created. I decided to make the navigation bar light blue because it goes well with pink also the font that was I used was very basic and not too small or big so that it is easy to use.
